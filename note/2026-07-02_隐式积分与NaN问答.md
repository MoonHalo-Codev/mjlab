# 隐式积分、qacc 与 NaN（问答整理）

> 日期：2026-07-02  
> 背景：阅读 mjlab 文档后，关于 Built-in / Explicit 执行器与数值积分的追问。

---

## 1. 简单例子：为什么是 \(m\ddot{q} = -k_p q - k_d \dot{q}\)？

因为假设 **期望位置、期望速度均为 0**（调节到原点）。

一般 PD：

\[
\tau = k_p (q_{\text{des}} - q) + k_d (\dot{q}_{\text{des}} - \dot{q})
\]

令 \(q_{\text{des}}=0,\ \dot{q}_{\text{des}}=0\) 即得 \(\tau = -k_p q - k_d \dot{q}\)。

RL 里 `use_default_offset=True` 时，policy 输出 0 对应 **默认站立姿态**，不是全零关节角。

---

## 2. 显式 vs 隐式（一阶直觉）

连续：\(m\ddot{q} = -k_p q - k_d \dot{q}\)

**显式**：阻尼用 \(\dot{q}_k\)

\[
\dot{q}_{k+1} = \dot{q}_k + \Delta t \cdot \frac{-k_p q_k - k_d \dot{q}_k}{m}
\]

**隐式**：阻尼用 \(\dot{q}_{k+1}\)

\[
m \frac{\dot{q}_{k+1} - \dot{q}_k}{\Delta t} = -k_p q_k - k_d \dot{q}_{k+1}
\]

解出：

\[
\dot{q}_{k+1} = \frac{m\dot{q}_k - \Delta t\, k_p q_k}{m + \Delta t\, k_d}
\]

分母 \(m + \Delta t k_d > 0\) → 阻尼项恒起稳定作用。

---

## 3. 为什么 \(k_d\) 大、\(\Delta t\) 大时显式容易炸？

状态 \(\mathbf{x}=[q,\dot{q}]^T\)，**显式欧拉**：\(\mathbf{x}_{k+1} = (I + \Delta t A)\mathbf{x}_k\)。

离散极点 = \(I+\Delta t A\) 的特征值。稳定 ⟺ 全在单位圆内（\(|\lambda|<1\)）。

一阶子问题 \(\dot{x}=-ax\)（\(a=k_d/m\)）：

- 显式：\(x_{k+1}=(1-a\Delta t)x_k\)，要 \(0<\Delta t<2/a\)
- \(k_d\) 大 → 允许的 \(\Delta t\) 更小；\(\Delta t\) 过大 → \(|1-a\Delta t|>1\) → 发散

隐式：\(x_{k+1}=x_k/(1+a\Delta t)\)，极点恒在 \((0,1)\) 内。

**Built-in + implicitfast 稳，Explicit PD 脆**：不是连续模型谁更真，而是 **离散极点/增益** 谁更安全。

---

## 4. mjlab 里 Built-in vs Explicit 代码路径

| | Built-in | Explicit (`IdealPdActuator`) |
|--|----------|------------------------------|
| `compute()` | 写 `position_target` 到 ctrl | Python 算 \(\tau=k_p e + k_d \dot{e}\) 写入 `<motor>` |
| MuJoCo 视角 | 原生 `<position>`，知 PD 结构 | 黑盒外力，不知 \(-k_d\dot{q}\) |
| 阻尼离散 | 可走 implicitfast 隐式 | 阻尼已烙进 \(\tau_k\)，等效显式 |

`BuiltinPdActuator`：`<position>` + `<velocity>` 配对，同时支持非零速度目标，D 项仍可隐式。

---

## 5. 为什么 Explicit 更容易 NaN？

因果链：

```
高 kp/kd 或 大 dt（显式阻尼）
  → q, qdot 放大
  → 接触/约束力极大
  → 求解器 qacc 异常
  → qpos/qvel/qacc/sensordata 出现 Inf/NaN
  → 观测 NaN → policy std 报错（FAQ 典型报错）
```

mjlab 检测：`qpos`, `qvel`, `qacc`, `qacc_warmstart`, `sensordata`（见 `NanGuard.detect_nans`）。

**NaN 常见吗？** 不是每跑必现，但 sim RL 里足够常见；MuJoCo Warp 仍在 beta。官方任务 + built-in 相对少；自改 env、explicit 高增益、极端 DR 会增多。工具：`nan_detection`（band-aid）、`nan_guard`（查根因）。

---

## 6. MuJoCo 一步在算什么？为什么要 qacc？

概念顺序：

```
ctrl + actuator 类型 → 广义力 τ（+ 接触约束）
  → 解 qacc（M·qacc + … = τ）
  → 积分：qvel、qpos 更新
  → forward：派生量、传感器
```

**qacc 不是速度更新后的多余量**，而是每步动力学要 **先解的未知量**，再用来更新状态。

---

## 7. 追问：隐式里 \(\dot{q}_{k+1}\) 已解出，还要 qacc 吗？

**不矛盾。** 两种写法是 **同一个隐式方程**：

**写法 A**：直接解 \(\dot{q}_{k+1}\)

**写法 B**：令 \(\ddot{q}_k=(\dot{q}_{k+1}-\dot{q}_k)/\Delta t\)，解 \(\ddot{q}_k\)，再 \(\dot{q}_{k+1}=\dot{q}_k+\Delta t\,\ddot{q}_k\)

等价形式（1D）：

\[
(m + k_d \Delta t)\,\ddot{q}_k = -k_p q_k - k_d \dot{q}_k
\]

代入 \(\dot{q}_{k+1}=\dot{q}_k+\Delta t\,\ddot{q}_k\) 即得写法 A 的公式。

```
        ┌──────────────────────────┐
同一方程 → │ 解 qdot_{k+1} 或 解 qacc │ → qvel 更新完 → 再更新 qpos
        └──────────────────────────┘
```

- **1D 教学**：直接解 \(\dot{q}_{k+1}\) 即可，qacc 只是定义 \(\ddot{q}=(\dot{q}_{k+1}-\dot{q}_k)/\Delta t\)
- **MuJoCo 多 DOF + 接触**：自然以 **qacc 为未知** 组装 \(M(q)\ddot{q}+\cdots=\tau\)；implicitfast 改的是求 qacc 的方程（有效质量矩阵含 \(\partial f/\partial\dot{q}\)）

**结论**：隐式里速度和 qacc 绑在一起；解其一即完成本步动力学，没有「速度算完又多余算一遍 qacc」。

## 8. 与实机控制的对比

| | 实机底层 | RL 仿真 |
|--|----------|---------|
| 周期 | 固定 1kHz 等 | physics dt 常 2–5ms + decimation |
| 数值问题 | 延迟、噪声为主 | 离散积分发散 → NaN |
| 选型 | — | Built-in 优先求 **训稳**；Explicit 用于特殊 actuator 模型 |
